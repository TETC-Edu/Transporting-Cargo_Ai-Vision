Find the cargo. Score the goal.

Two sports balls are somewhere out on the field — and you don't know exactly where. Your robot's AI Vision Sensor is the only thing that can spot them, line up the goal, and kick both through before the clock runs out.

2 sports balls 1 goal 1 AI Vision Sensor

What you're doing

Here's the twist: the balls move. You can't measure a distance and hard-code it, because the next round they'll be somewhere else. So instead of telling the robot where to go, you teach it to look — the AI Vision Sensor spots a ball, tells you which way to turn, and you drive in and kick. You'll solve the same challenge three times — once in Blocks, once in Switch, and once typing real Python — and each round you'll move the balls to prove your code really sees.

The arc

You'll solve the same challenge three times. Each level adds one new thing:

  • Level 1 — Blocks. Use the AI Vision blocks to find one ball, line up the goal, and kick it in.
  • Level 2 — Switch. Convert your blocks to see the Python, and read how the vision numbers steer the robot.

Step 1: Sketch your plan

Grab an Expo marker. On the field mat, sketch:

  • The goal at the top: AprilTag 0 between the two orange barrels.
  • A START mark where the robot begins, facing the field.
  • An arrow showing how the robot will spin to search, then drive in once it spots a ball.

You can't draw an X for the ball — you don't know where it'll be. That's the whole point. Your plan is about how the robot looks, not where things are.

The mat is dry-erase — the marker comes right off when you're done with the level.

Step 2: Turn on AI Vision and test it

In VEXcode AIM, open the AI Vision setup. Make sure it's detecting the Sports Ball and AprilTag. Then drive by hand and watch the camera view:

  • When the ball is in view, the sensor says it exists. When it's not, it doesn't. Your code will check that.
  • The sensor also reports a bearing — how far left or right the ball is from straight ahead. That tells the robot which way to turn.

Go slow. If you drive too fast, the camera can't keep up and the ball flickers in and out.

Step 3: Code it in Blocks

Open VEXcode AIM in Blocks mode. Build a stack that looks something like this:

  • Look for the Sports Ball with the AI Vision block.
  • Turn a little at a time until the ball is in view (use the ball's bearing to pick left or right).
  • Drive forward to get close to the ball.
  • Look for AprilTag 0, turn to line up the goal, then Kick.

Run it. The ball won't be in the same spot twice, so test it a few times with the ball in different places.

Switch
Your task

Move the ball somewhere new. Convert your blocks to Switch mode to see the Python underneath. Your job here isn't to fix numbers — it's to read the code and understand how .exists and .bearing make the robot react to wherever the ball actually is.

Heads up — Your ball won't be in the same spot twice. There's no number you can hard-code to win — the robot has to look. Notice how almost nothing in the code is a fixed position. That's on purpose.
First: How to read Python

When you turn your blocks into Switch, you'll see lines of Python like this for the first time. Don't panic. Every line in this lesson follows the same shape:

robot.move_for(450, 0)
  • Whorobot — who are you talking to? The robot.
  • .The dot means "go inside" or "use your..." — think of it as opening a toolbox.
  • Whatmove_for — what do you want it to do? Move.
  • (Open parenthesis — here come the details.
  • Details450, 0 — the specifics. 450 millimeters, 0 degrees. Commas separate them. Order matters.
  • )Close parenthesis — done.

In English: "Hey robot, use your move — 450 mm, 0 degrees."

Try it on three more

robot.turn_for(LEFT, 90)
Who: robot  ·  What: turn_for  ·  Details: LEFT, 90 degrees
"Hey robot, use your turn — left, 90 degrees."
robot.kicker.kick(HARD)
Who: robot.kicker (the kicker inside the robot — two dots = two toolboxes deep)  ·  What: kick  ·  Details: HARD power
"Hey robot, open your kicker toolbox, use the kick — hard power."
wait(0.5, SECONDS)
Who: nobody — this one isn't aimed at the robot  ·  What: wait  ·  Details: 0.5, SECONDS
"Pause for half a second."
Two dots? That just means you're going deeper. robot → open the robot → kicker → open the kicker toolbox → kick → use the kick tool. It's nested boxes.

Read any line

  1. Find the dots. Everything before the last dot is the Who. The word after the last dot (before the parenthesis) is the What.
  2. Look inside the parentheses. Those are the Details. Commas separate them. Order matters — the first detail is always the first thing the command needs to know.
  3. Say it out loud. "Hey [Who], do [What] with [Details]." If it sounds right in English, you probably read it right.

Step 3: Type the find-and-score code

        
The shape is: turn on tag detection, spin until you see a ball, turn toward it, drive in, then kick. Here's a starting point — type it, then tune the numbers for your robot:

        


# See the goal's AprilTag too
robot.vision.tag_detection(True)

# 1. Spin slowly until a ball is in view
ball = robot.vision.get_data(SPORTS_BALL)
while not ball[0].exists:
    robot.turn_for(RIGHT, 15)
    ball = robot.vision.get_data(SPORTS_BALL)

# 2. Turn toward the ball, then drive in
if ball[0].bearing > 0:
    robot.turn_for(RIGHT, ball[0].bearing)
else:
    robot.turn_for(LEFT, -ball[0].bearing)
robot.move_for(300, 0)

# 3. Kick it through the goal
robot.kicker.kick(HARD)

        

        
Want it cleaner? Before the kick, look for the goal with robot.vision.get_data(TAG0) and turn toward its bearing the same way you did for the ball — so you're aimed at the goal, not just facing the ball.

          Your task
          
The real challenge wants both sports balls in the goal. You could copy your Level 3 code and paste it twice — or you could wrap it in a loop and run it as many times as you like. Same find-and-score code, scoring two balls instead of one.

        


        

        

          
1. Spot the pattern

          
Look at the code you wrote in Level 3. To score a second ball, you'd do the exact same thing again: search, turn, drive, kick. What's the same every time? What's different?

          
Talk it through with your partner before reading on.


          

            The answer —
            The shape is identical every time: search, turn toward the ball, drive, kick. Almost nothing is different — the vision figures out the actual angles for you. That's exactly what a loop is for.
          

        


        

          
2. What's a variable?

          
A variable is a name for a value. Instead of writing HARD or 15 in several places, you name it once and use the name. Change it in one spot and every use updates.

          


# Before (value typed inline)
robot.kicker.kick(HARD)

# After (with a variable)
power = HARD
robot.kicker.kick(power)

          

          
Small thing now — but if you kick in three places and want to try SOFT, you change one line instead of three.

        


        

          
3. What's a loop?

          
A loop means: do this thing more than once. Instead of writing the find-and-score code twice (once per ball), write it once and tell the robot "do this 2 times."

          
range(2) is just "count out 2 turns." The robot runs the loop body once for each one.


          


# Do something 2 times
for i in range(2):
    robot.screen.print("Ball")

          

          
This prints twice. Change 2 to 5 and it prints five times. The loop does the repeating for you.

        


        

          
4. Build it

          
Now put it together. Take your Level 3 find-and-score code and wrap it in a for loop that runs twice.

          
Step by step:

          
    
            
  1. Start for i in range(2):
    2. 
            
  2. Indent your whole find-and-score block underneath it.
    3. 
            
  3. After the kick, back up or turn away so the next search finds the other ball.
    4. 
          


          
Here's the shape — fill in your own numbers:

          


robot.vision.tag_detection(True)
power = HARD

# Score two balls
for i in range(2):
    # search until a ball is in view
    ball = robot.vision.get_data(SPORTS_BALL)
    while not ball[0].exists:
        robot.turn_for(RIGHT, 15)
        ball = robot.vision.get_data(SPORTS_BALL)

    # turn toward it and drive in
    if ball[0].bearing > 0:
        robot.turn_for(RIGHT, ball[0].bearing)
    else:
        robot.turn_for(LEFT, -ball[0].bearing)
    robot.move_for(300, 0)

    # kick it through the goal
    robot.kicker.kick(power)

          

          
Tune the search step and drive distance for your robot. Don't change the loop structure — just the numbers inside.

        


        

          
5. The payoff

          
You wrote the find-and-score code once. The loop ran it twice. Want to score five balls? Change range(2) to range(5). That's the only edit.


          

            That's the whole point —
            Variables and loops aren't fancy computer-science magic. They're what you do when you notice you're about to copy-paste the same code. You had the problem first — two balls, one block of code. Now you have the fix.
          

        


        

          
Stuck?

          


            

              

                
?My loop only runs once, or not at all.

                
              

              

                

                  Try first
                  Check the number in range(). range(2) runs twice. range(1) runs once. For both balls you want range(2).
                

                

                  Then this
                  Check indentation. Every line inside the for must be indented the same (use 4 spaces). Lines that aren't indented run after the loop, not inside it.
                

                

                  Approach
                  Put robot.screen.print("loop") as the first line inside the loop. Now you can see how many times it runs on the robot screen. If you see it twice, the loop is fine and the bug is somewhere else.
                

                

              

            


            

              

                
?It scores the same ball twice.

                
              

              

                

                  Try first
                  After the kick, the first ball is gone but the robot is still facing where it was. The second time through, it sees the leftover... nothing, or re-finds the same spot. It needs to move on before searching again.
                

                

                  Then this
                  Add a turn or a short back-up at the end of the loop, so the next search starts from a different view. Now the spin-to-search finds the other ball.
                

                

                  Approach
                  Print which time through you are: robot.screen.print(i) at the top of the loop. Watch the robot. If it kicks toward the same corner twice, that's your clue it never turned away.
                

                

              

            


            

              

                
?IndentationError.

                
              

              

                

                  Try first
                  Python reads spacing. Everything inside the for loop has to be indented the same amount — use 4 spaces, not tabs.
                

                

                  Then this
                  You have a loop inside a loop here: the while search sits inside the for. The while lines need to be indented further still. Count your spaces carefully.
                

                

                  Approach
                  Line up the start of each line with your eyes. Lines that belong together start at the same column. If one is off by a space, fix it and run again.
